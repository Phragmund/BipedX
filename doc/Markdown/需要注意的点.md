# 需要注意的点

我结合咱们之前敲定的“**1kHz高频姿态环**”、“**平衡车+丝杆变形**”以及“**外骨骼整机联动**”这三大硬性需求，对这版核心功能清单和流程图进行了“压力测试”。

如果是初次接触这类复杂系统，最容易踩的坑是**“时序阻塞”\**和\**“状态混乱”**。以下是我揪出的几个关键漏洞和必须补充的模块，建议直接加入你们的系统设计中：

### 🚨 1. 架构级致命伤：千万别把“平衡控制”放在 `while(1)` 里！

很多新手画流程图，会顺理成章地画成：`初始化 -> while(1) {读IMU -> 算PID -> 发电机指令}`。

- **漏洞**：在主循环里做控制，一旦被外骨骼通信拆包、串口打印或者变形机构的逻辑卡住几毫秒，1kHz的姿态环就会剧烈抖动，甚至直接抽搐摔倒。
- **修改方案**：必须采用 **“定时器中断 (Timer Interrupt) + 主循环后台”** 的前后台架构。
  - **高频中断 (最高优先级，1ms触发一次)**：只干三件事——通过SPI读IMU数据 -> 跑姿态解算和PID控制算法 -> 通过CAN把电流指令发给无刷电机。
  - **主循环 `while(1)` (低优先级)**：处理非实时任务——解析外骨骼发来的长串通信包、控制变形推杆电机、检测电池电压、控制LED状态指示灯。

### 🔄 2. 核心功能补充：“轮-脚”切换的平滑状态机 (FSM)

从“轮子”变成“脚”不是一瞬间的事，这个过程在程序里不能只有一个标志位，必须是一个**有限状态机**。

- **补充建议**：在流程图的核心逻辑中，加入明确的四种状态：
  1. **轮式平衡态 (Wheel Mode)**：正常跑平衡车双环PID。
  2. **准备变形态 (Pre-Transform)**：接收到外骨骼指令，首先**刹车减速至停止**，锁死轮毂电机位置环。
  3. **变形进行态 (Transforming)**：驱动推杆丝杆电机。此时整机物理构型在变，重心在变，**必须动态调整平衡PID的参数**（或者如果有人穿着，此时交由人腿支撑，关闭平衡控制）。
  4. **外骨骼随动态 (Leg Mode)**：推杆到达限位开关后停止，此时轮子变成了脚底板，电机切换为“零力矩拖动模式”或“外骨骼步态协同模式”。

### 🛡️ 3. 必须追加的模块：安全与异常守护进程 (Safety Watchdog)

外骨骼是穿在人身上的，安全级别比普通的寻迹小车高几个数量级。清单里必须加上以下异常处理机制：

- **倾倒保护**：读到IMU的Pitch或Roll角度超过±45°（意味着摔倒了），必须**立刻切断**轮毂电机输出（发0电流），防止电机疯转伤人。
- **通信失联保护**：如果超过 50ms 没收到外骨骼主控的心跳包，应该触发减速悬停，而不是继续执行上一个指令冲出去。
- **推杆堵转保护**：变形用的丝杆电机必须有过流检测或限位开关硬件中断，万一卡住了能立刻停下，防止烧板子。

### ⚙️ 4. 开机流程补充：上电静止与校准

- **补充建议**：流程图的“初始化”环节之后，不能直接进入控制。必须加一个 **“等待IMU零偏校准”** 的环节。每次开机前几秒，系统需要静止在地面上，采集几百个IMU数据计算陀螺仪的零点漂移误差。如果不做这一步，平衡车跑着跑着就会往一边倒。

### 📂 一、 规范的工程目录结构 (基于CubeMX生成器)

为了避免所有代码全挤在 `main.c` 里导致后期无法维护，我们采用“前后台系统 + 模块化”的结构：

- **`Core/Src/main.c`**：主程序入口，负责硬件初始化、开机校准和低频后台任务调度。
- **`Core/Src/stm32f1xx_it.c`**：中断服务函数集中营，**1kHz姿态控制环**的命脉就在这里的定时器中断里。
- **`User/Src/state_machine.c`**：核心状态机逻辑，处理“轮-脚”切换。
- **`User/Src/control_pid.c`**：电机闭环控制算法（直立环、速度环、推杆位置控制）。
- **`User/Src/bsp_imu.c`**：SPI读取IMU数据及滤波解算底层驱动。
- **`User/Src/task_comm.c`**：与外骨骼上位机的CAN/串口通信及安全看门狗（Watchdog）。

### 💻 二、 核心伪代码解析

#### 1. `main.c` —— 后台主循环 (非实时任务)

主循环主要处理优先级不高的系统级任务，绝对不要在里面放任何阻塞式（`Delay`）的控制逻辑。

C

```
/* Core/Src/main.c */
#include "main.h"
#include "state_machine.h"
#include "task_comm.h"

int main(void) {
    // 1. 硬件外设初始化 (CubeMX自动生成)
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_SPI1_Init();   // 初始化IMU通信
    MX_CAN_Init();    // 初始化轮毂电机及外骨骼通信
    MX_TIM6_Init();   // 启动1kHz定时器，用于高频控制环
    
    // 2. 开机静止与IMU零偏校准 (关键防护！)
    LED_SetStatus(LED_CALIBRATING);
    IMU_WaitAndCalibrate(); // 采集1000组数据计算零点漂移
    LED_SetStatus(LED_READY);
    
    // 3. 启动硬件定时器中断和外设
    HAL_TIM_Base_Start_IT(&htim6); // 开启1kHz中断
    CAN_Start_Receive_IT();        // 开启CAN接收中断

    // 4. 后台低频主循环 (Background Loop)
    while (1) {
        // 任务A：解析外骨骼发来的长通信包 (非实时)
        if (Comm_HasNewPacket()) {
            Comm_ParseData();
        }
        
        // 任务B：安全守护进程 (Watchdog) - 50ms跑一次
        if (Check_Timeout(50)) {
            Safety_Watchdog_Task(); 
            // 检查内容：电池欠压？外骨骼通信失联？电机温度过高？
        }
        
        // 任务C：状态指示灯刷新
        LED_BlinkTask(GetCurrentState());
    }
}
```

#### 2. `stm32f1xx_it.c` —— 前台高频中断 (1kHz 实时控制)

这里是系统的最高优先级区域。每1毫秒（1ms）强制打断主循环执行一次，必须保证代码精简，运行时间不能超过1ms。

C

```
/* Core/Src/stm32f1xx_it.c */
#include "state_machine.h"
#include "bsp_imu.h"
#include "control_pid.h"

// 假设使用TIM6作为1kHz控制周期的定时器
void TIM6_IRQHandler(void) {
    HAL_TIM_IRQHandler(&htim6); // 清除中断标志位
    
    // 1. 读传感器：通过SPI极速读取IMU数据并滤波
    IMU_Data_t imu_raw = IMU_Read_SPI();
    Attitude_t posture = IMU_KalmanFilter(imu_raw); // 获取当前Pitch, Roll角度
    
    // 2. 硬件级异常拦截 (倾倒保护)
    if (posture.pitch > 45.0f || posture.pitch < -45.0f) {
        Motor_Send_CAN_Current(0, 0); // 紧急切断轮毂电机输出
        SetSystemState(STATE_EMERGENCY_STOP);
        return; // 退出当前控制周期
    }

    // 3. 运行核心状态机逻辑 (根据状态决定控制算法)
    StateMachine_Update(posture); 
}
```

#### 3. `state_machine.c` —— 轮脚转换的灵魂

这部分决定了系统在当前毫秒应该干什么。

C

```
/* User/Src/state_machine.c */

typedef enum {
    STATE_WHEEL_MODE = 0,  // 轮式平衡态
    STATE_PRE_TRANSFORM,   // 准备变形态
    STATE_TRANSFORMING,    // 变形进行态
    STATE_LEG_MODE,        // 外骨骼随动态(脚模式)
    STATE_EMERGENCY_STOP   // 紧急停止态
} SystemState_e;

SystemState_e current_state = STATE_WHEEL_MODE;

// 被1kHz定时器中断调用的核心控制分配器
void StateMachine_Update(Attitude_t posture) {
    switch(current_state) {
        
        case STATE_WHEEL_MODE:
            // 运行倒立摆直立环 + 速度环 PID
            float torque = PID_Balance_Calc(posture.pitch, Target_Angle);
            torque += PID_Speed_Calc(Motor_GetSpeed(), Target_Speed);
            Motor_Send_CAN_Current(torque, torque); // 发送给大疆轮毂电机
            
            // 收到外骨骼切换指令？
            if (Check_Transform_Cmd()) {
                current_state = STATE_PRE_TRANSFORM;
            }
            break;
            
        case STATE_PRE_TRANSFORM:
            // 变身前置动作：刹车，速度环目标设为0，等待完全停稳
            PID_Speed_Calc(Motor_GetSpeed(), 0);
            if (Motor_GetSpeed() == 0 && abs(posture.pitch) < 2.0f) {
                current_state = STATE_TRANSFORMING;
            }
            break;
            
        case STATE_TRANSFORMING:
            // 锁死轮毂电机位置，防止滑动
            Motor_Hold_Position();
            // 驱动推杆丝杆电机伸缩
            LinearActuator_SetPWM(TARGET_POSITION);
            
            // 检查丝杆是否触碰限位开关
            if (LinearActuator_IsLimitReached()) {
                LinearActuator_SetPWM(0); // 停机自锁
                current_state = STATE_LEG_MODE;
            }
            break;
            
        case STATE_LEG_MODE:
            // 完全切换为外骨骼脚底板模式
            // 轮毂电机进入零力矩拖动（发0电流）或协同助力模式
            Motor_Send_CAN_Current(0, 0); 
            // 持续向外骨骼上位机报告当前脚部受力与姿态
            Comm_ReportToExoskeleton(posture);
            break;
            
        case STATE_EMERGENCY_STOP:
            // 瘫痪模式，所有致动器断电
            Motor_Send_CAN_Current(0, 0);
            LinearActuator_SetPWM(0);
            break;
    }
}
```

### 💡 秘书的执行建议
