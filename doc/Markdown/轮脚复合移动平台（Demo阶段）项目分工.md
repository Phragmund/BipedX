## 一、总体开发原则与工作流

为确保项目推进的高效性与样机的安全性，全体成员需严格遵守以下原则：

1. **安全T0原则（一票否决制）：** 作为载人设备，任何工况下的断电、急停，均不得导致结构溃缩或失控。所有机械自锁设计与软件失控保护逻辑拥有最高优先级。
2. **接口先行原则：** 严禁无边界开发。所有详细设计展开前，必须由总布置（ME1）确认各模块的空间包络与电气接口。各组需在规定“边界”内进行设计。
3. **去中心化采购机制（谁设计、谁提单、谁采购）：** 为提高效率，取消总师单一审核制。全组共用一个在线BOM表格（腾讯文档/飞书），各组成员根据自己的设计直接将所需标准件（螺丝、轴承、电子元器件等）填入表格并自主推进采购。
4. **Demo导向原则：** 本阶段优先“验证功能跑通”与“保障核心安全”，暂不追求量产级别的外观工业设计与极致成本压缩。



## 概述：

**1. 产品定义** 本项目研发一款具备双模式切换功能的穿戴式智能移动平台（轮脚复合机器人）。该平台旨在通过机械机构与自动控制算法的结合，实现个人在不同地形下的高效代步与越障。

**2. 核心功能模式**

- **平衡模式（自平衡滑行）：** 利用内置陀螺仪解算姿态，通过电机驱动外侧轮组实现动态平衡。用户通过身体重心或遥控器控制平台的前进、后退与转向，适用于平整路面的高效滑行。
- **行走模式（多点支撑）：** 通过机械起落架机构将轮式支撑切换为多点几何支撑。该模式下平台不再依赖动态平衡算法，可适应楼梯、碎石等复杂地形的步进需求。

**3**.系统深度集成**T0级安全防护**（失控自锁/自动急停）与**能量回收逻辑**，并在物理设计上利用**便携卡扣**实现双足吸合、一拎即走，同时预留**外骨骼动力接口**，为未来的人机协同进化预留了无限可能。

## 二、机械工程组（ME）任务下达

### ME1：项目总布置与便携设计（由 杨滨瑞 负责）

**【职责定位】：** 项目统筹、宏观边界定义与交互巧思设计。

- **整机Layout与边界控制：** 定义整机最大尺寸限制（长、宽、高），并在3D空间中划定“变形机构专属区”与“电池主控专属区”，输出整机骨架简图。
- **便携交互系统设计：** 负责设计双脚吸合/卡扣机构，确保在非使用状态下，两只鞋能够合二为一，便于用户单手提携（此部分可自由发挥结构巧思）。
- **信号环境与外骨骼宏观接口：**宏观划定天线与遥控接收器的合理布置区域（避开金属屏蔽区）；敲定与外骨骼通信接口的物理形态与空间坐标。

### ME2：核心机械与运动学攻坚（由 辛昌昊 负责）

**【职责定位】：**负责整机最核心的机械变形逻辑与结构可靠性。

- **轮-脚切换机构设计与触地组件：** 独立设计“起落架”收放结构；设计起落架末端的减震脚垫。**关键指标：必须具备纯机械自锁功能。**确保在“脚模式”下，仅靠机械结构即可稳定支撑，无需电机持续输出扭矩对抗冲击，且触地组件具有类似脚掌的缓冲能力。
- **强度校核（FEA）：**针对变形机构关键受力连杆出具有限元分析报告，验证在75kg载重及动态冲击下，机构无屈服断裂风险。
- **防干涉验证：**在动态仿真中排查运动干涉；预留微动开关安装位，用于向主控反馈“腿部已物理锁死”的状态。

### ME3：底盘动力与硬件集成（由 张竞文 负责）

**【职责定位】：** 负责主承重底盘的设计、传动系统搭建及物理安全设施。

- **主承重底盘与外骨骼硬点：** 设计“后侧走入式”主体框架（参考耐克懒人鞋穿脱方案）；负责脚踏区域的设计（需考虑防滑、**固定绑带**的安装孔位，参考轮滑鞋或者耐克回到未来等设计）。

- **重点：在底盘/鞋套上设计高强度的法兰孔、插销或滑轨等机械硬点（Hardpoints），用于未来物理锁死并承载外骨骼。**

- **传动组件：** 完成“侧轮+链轮”安装座设计、链条张紧机构及必要时用链条防护罩封装。

  ​	计算出在30厘米高台上跃下时，单根车轴承重约3000N，考虑铝合金管掏空外径须达17.5mm。

  ​	在强度能够保证的前提条件下，在轮毂外加上**悬浮轮毂盖**或**静止轮毂罩**（参考劳斯莱斯的轮毂车标）。

- **“三口一关”全硬件收纳与线束规划：** 在底盘合理开孔固定：手动安全总开关、遥控信号接收器、外骨骼动力/通信接口（和对方机械工程师A对接拓展接口和D对接固定方式），并规划内部走线槽，电池需考虑可更换设计。

  须达到不飞线，方便与外骨骼进行机械连接与断开的效果。

  

  ****单边轮脚重量最好不超过5kg。
  
  【安全底线】物理测试保护架搭建：** 负责在首次载人上电调试前，搭建简易的门型吊架，或采购配备承重安全衣与悬挂绳，**严禁无物理保护直接上车调参。**

## 三、 电控与嵌入式组（EE）任务下达

### EE1：硬件开发与底层驱动（由 张宁颖 负责）

**【职责定位】：** 负责能源管理、底层信号收发、硬件级保护及状态反馈硬件。

- **主控板绘制与能量回收：** 绘制集成STM32、IMU与CAN总线的控制板。必须设计过压保护与电流回灌路径（如大电容阵列），安全吸收电机制动产生的反向电动势。

  ​	考虑到陀螺仪和单片机总线间的带宽为1kHz，同时考虑控制板的散热设计。

- **多电源协同电路：** 设计主电池与外骨骼备用电池的并联/切换保护电路（与对方嵌入式工程师A对接）。

  ​	电池续航须达到一个小时或续驶里程达到25公里。

- **声光反馈硬件与遥控器部署：**
  
  - 在主板上集成RGB状态指示灯。
  
    加装电量指示灯4格或5格，示廓灯夜路安全行驶，转向时转向灯闪烁。
  
  - 负责遥控器与接收机的硬件选型（如AT9S或小枪控）、通信协议匹配（SBUS等）及硬件对频。（具体情况与黄钰翔交接）
  
- **底层驱动与硬件保命：** 封装大疆电机CAN通信；实现硬件级过压/欠压中断：当母线电压超越安全阈值时，自动物理切断动力回路。

## 四、 算法与控制组（AE）任务下达

### AE1：运动控制与状态决策（由 张哲灵 负责）

**【职责定位】：** 负责姿态平衡、复杂运动逻辑、整机状态机与人机交互逻辑。

- **复合运动控制与遥控映射：** 基于滤波算法融合IMU数据解算姿态。解析遥控器通道指令，实现轮/脚切换；实现通过角度传感器进行前进与差速转向。

- **能量回收策略：** 检测到下坡或减速指令时，根据当前母线电压动态调整电机电流环，实现安全能量回收与平滑刹车。

- **大管家状态机与异常仲裁：** 编写系统状态流转逻辑（待机 -> 自检 -> 平衡 -> 变形 -> 行走）。
  
  - 变形指令生效先决条件：手动开关=ON 且 车速=0。
  
  - **T0级异常急停：** 遇倾角>60°、电压异常或断联，立刻切断动力。
  
    #### ***设定低电量阈值，低于阈值时功率逐级降低至0保证安全性，避免断电突然摔车***
  
    **加减速度平稳**
  
- **声光报警逻辑开发：** 将底层状态转化为人机交互语言。例如：自检通过（绿灯/滴一声）、低压报警（红灯闪烁）、姿态失控/未锁死强行启动（长鸣报错）。（需与张宁颖协作）

## 五、 软硬件协同接口标准（ICD）

各模块无缝对接依据，全员需严格执行：

### 5.1 机械与电气物理接口

- **机械连接：** 模块间固定统一采用 **M4** 标准螺纹孔。
- **动力接插件：** 24V主电源使用 **XT60**；外骨骼备用接口使用 **XT60-F**。
- **信号接插件：** 板载通信使用 **GH1.25**；遥控器接入端预留防呆插针。

### 5.2 软件通信协议（软硬解耦结构体）

EE1与AE1通过共享以下数据结构体进行底层数据与顶层算法的交互：

C

```
typedef struct {
    // ---- 由 EE1 负责采集并更新 (硬件输入层) ----
    float imu_pitch;       // 滤波后的俯仰角
    float imu_gyro_y;      // Y轴角速度
    float motor_speed_L;   // 左轮实时转速
    float motor_speed_R;   // 右轮实时转速
    float bus_voltage;     // 实时母线电压 (防烧板与能量回收判断依据)
    int   remote_ch_A;     // 遥控器模式切换通道值
    int   remote_ch_B;     // 遥控器速度通道值
    int   hw_switch_state; // 手动安全开关状态 (1:ON, 0:OFF)
    
    // ---- 由 AE1 负责计算并写入 (算法输出层) ----
    float target_torque_L; // 计算出的左轮目标扭矩
    float target_torque_R; // 计算出的右轮目标扭矩
    int   energy_regen_en; // 能量回收使能标志 (0:禁用, 1:启动)
    int   fsm_state;       // 系统状态机状态 (用于控制RGB灯色)
    int   buzzer_cmd;      // 蜂鸣器控制指令 (0:关, 1:短鸣, 2:急促, 3:长鸣)
} Robot_State_t;
```
